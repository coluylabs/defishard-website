## Typography

### Font Families

| Type | Font | Usage |
|------|------|-------|
| **Sans-serif** | Inter | Headings, body text |
| **Monospace** | JetBrains Mono | Labels, technical text, buttons, code |

### Font Styles

- **Headings:** Bold, tight tracking (-0.02em)
- **Body:** Regular weight, relaxed line-height
- **Labels:** Monospace, uppercase, extra letter-spacing (0.1em)
- **Buttons:** Monospace, uppercase, semibold, small size

### Hierarchy

- **XL Heading:** 48-72px, Bold, White
- **Large Heading:** 24-36px, Bold, White
- **Body:** 16-18px, Regular, White or Slate
- **Labels:** 12px, Mono, Uppercase, Slate
- **Technical IDs:** 12px, Mono, Slate Border color

## Visual Style

### Background Treatment

- **Primary:** Solid deep charcoal (`#0F1115`)
- **Gradient:** Linear gradient from `#0F1115` to `#161920`
- **Grid Pattern:** Subtle lime grid lines at 2-3% opacity
- **Scan Lines:** Horizontal repeating lines at 1-2% opacity (retro-tech feel)

### Card & Surface Design

- **Glass Morphism:** Semi-transparent backgrounds with backdrop blur
- **Borders:** 1px solid slate (`#2D3340`), lime on hover
- **Border Radius:** 8px for cards, 4px for buttons, 24px for large containers
- **Shadows:** Flat technical shadow (0 4px 0 0 rgba(0,0,0,0.2))

### Hover States

- Border color transitions to lime (`#CCFF00`)
- Subtle lift effect (-1px translateY)
- Enhanced shadow depth

## UI Components Style

### Buttons

**Primary Button:**
- Background: Electric Lime (`#CCFF00`)
- Text: Black (`#000000`)
- Style: Flat with bottom shadow, uppercase monospace text
- Hover: Slightly darker lime, increased shadow

**Secondary Button:**
- Background: Transparent
- Border: Slate (`#2D3340`)
- Text: White
- Hover: Border and text turn lime

### Cards

- Background: `#161920` or glass effect
- Border: 1px `#2D3340`
- Padding: 24-32px
- Corner accent: Small lime lines at corners (optional)
- Technical ID marker in top-right corner

### Badges/Tags

- Background: Lime at 10-20% opacity
- Border: Lime at 20-30% opacity
- Text: Lime, uppercase, monospace
- Rounded pill shape

## Iconography & Graphics

### Style Guidelines

- **Line weight:** 2px stroke
- **Style:** Outlined, geometric, minimal
- **Color:** White or Slate, Lime for emphasis
- **Size:** 20-32px for UI, larger for hero sections

### Key Visual Elements

1. **Shield** - Security, protection
2. **Smartphone** - Mobile authenticator
3. **QR Code** - Connection/pairing
4. **Lock** - Encryption
5. **Key fragments** - MPC shares (never show whole key)
6. **Circuit patterns** - Technical/crypto aesthetic

## Animation & Motion

### Principles

- Subtle, professional movements
- Ease-out timing functions
- 200-400ms duration for UI interactions
- Floating/breathing animations for ambient elements

### Common Animations

- **Fade In:** 400ms ease-out
- **Slide Up:** 400ms with 20px travel
- **Pulse/Glow:** 2-3s infinite for highlights
- **Float:** 6s infinite ease-in-out for decorative elements

## Do's and Don'ts

### Do's ✓

- Use high contrast between lime and dark backgrounds
- Keep designs clean and minimal
- Use monospace fonts for technical elements
- Show key "fragments" or "shares", never complete keys
- Emphasize security and precision
- Use subtle animations for polish

### Don'ts ✗

- Don't use bright/light backgrounds
- Don't show complete private keys or seed phrases
- Don't use cluttered or busy designs
- Don't use rounded, playful aesthetics
- Don't use gradients other than lime-to-white or dark-to-darker
- Don't use colors outside the defined palette
